File Watcher Utilities

File Watcher Utilities project provides file system/directory monitoring utilities with loggin and task processing support. The applications have multiple configuration options. Source code libraries or binaries can be used to create a custom file system monitor.

Applications

File Watcher Utilities currently contains multiple applications with different type of user interface. The applications can be used together or seperately. File Watcher Simple can be used to create XML configuration files for other applications. It is also the recomended option for editing and testing any configurations. Other applications support only manual editing of the XML configuration files.

  • File Watcher Console (command line version)
  • File Watcher Simple (graphical user interface version)
  • File Watcher Windows Service (windows service version)

Extensibility

File Watcher Utilities applications doesn't support plugins because the variety of applications and threading issues. Instead the File Watcher Utilities applications support out of process type of extensibility. The File Watcher Utilities applications can consume Windows Communication Foundation services (typically web services) that use the File Watcher Utilities specific service contracts. The service contracts allow developers to create file streaming and file change information storing type of services.

System requirements

  • Microsoft Windows 2000, Windows 2003 or Windows XP Professional or Home Edition, Windows Vista, Windows 2008, Windows 7 or later.
  • Microsoft .NET Framework 4.0 or later or Microsoft .NET Framework 3.0 (Service Pack 2) or Microsoft .NET Framework 2.0 (Service Pack 2)1
  • Applications can be compiled and run with Mono on other operating systems. For more details see Supported Platforms - Mono.

1Applications compiled with Microsoft .NET Framework 2.0 cannot call Windows Communication Foundation services.
